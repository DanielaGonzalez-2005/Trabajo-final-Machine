Planteamiento del Problema Geopolítico

La estabilidad de Oriente Medio y el mercado energético mundial dependen críticamente de las dinámicas del conflicto entre Irán, Israel y los Estados Unidos. Este proyecto diseña un sistema de alerta analítico para clasificar diariamente el nivel de confrontación.

Pregunta de Machine Learning:
¿Es posible clasificar el nivel de escalada del conflicto Irán–Israel–EE.UU. en una ventana país-día usando señales mediáticas, lingüísticas, económicas y de eventos físicos de fuentes abiertas?

Dimensiones OSINT

  • Dimensión Física: 29,811 eventos detallados extraídos de ACLED (combates, bombardeos y bajas) georreferenciados en la región.
  • Dimensión Mediática: El tono diario promedio global y la intensidad noticiosa monitoreada por GDELT.
  • Dimensión Económica: Precios internacionales de petróleo Brent (Yahoo Finance) como proxy de incertidumbre macro.

Análisis de Sentimiento Semántico

Utilizamos el modelo de lenguaje de alto rendimiento all-MiniLM-L6-v2 para traducir los titulares noticiosos a vectores densos (embeddings) de 384 dimensiones. Esto nos permite evaluar la similitud semántica contra plantillas predefinidas de escalada (Baja, Media y Alta) mediante similitud coseno, evitando los sesgos de negación del conteo tradicional de palabras.

Análisis Exploratorio de Datos (EDA)

El análisis exploratorio sobre los 2,592 registros del dataset arrojó hallazgos clave sobre el comportamiento de las variables de conflicto:

Desbalance del Target

La variable objetivo target posee una distribución asimétrica: 52.5% Baja Escalada, 32.1% Media Escalada y 15.4% Alta Escalada, requiriendo estrategias de balanceo.

Volatilidad del Brent

Los precios del barril Brent demuestran desviaciones y retornos más extremos durante días catalogados en Alta Escalada (2), validando su sensibilidad a las hostilidades físicas.

Focalización Física

ACLED muestra concentración de eventos de violencia armada en Israel e Irán, mientras que las tensiones en EE.UU. se canalizan principalmente de forma lingüística en GDELT.

Correlación de Variables: Las métricas de embeddings de titulares (score_alta_prom y pct_alta_emb) muestran una fuerte correlación lineal positiva con el target, mientras que el tono de GDELT (tono_gdelt) muestra una correlación negativa esperada (tonos muy negativos corresponden a confrontaciones).

Pipeline de Preprocesamiento y Validación

Antes de entrenar los clasificadores, aplicamos un riguroso pipeline para asegurar la reproducibilidad y evitar el sobreajuste:

1
Codificación

Transformamos el actor (pais) de texto a variables indicadoras numéricas binarias (One-Hot Encoding).

2
Validación Temporal

Dividimos cronológicamente: Train (2024-2025: 2,193 días) y Test (2026: 399 días), evitando lookahead bias.

3
Estandarización

Normalizamos las variables continuas con StandardScaler ($\mu=0$, $\sigma=1$), clave para KNN y modelos lineales.

4
Balanceo SMOTE

Generamos muestras sintéticas de entrenamiento para equiparar la distribución de clases (3,414 registros balanceados).

Comparación de Modelos y Evaluación

Detección del Target Leakage: Las columnas escalada_acled, tono_gdelt y sentimiento_medio componen la definición matemática del target. Para entrenar un modelo que sea verdaderamente predictivo y útil ante nuevos datos, las excluimos de las variables predictoras (Escenario B).
