Bitácora Técnica del Proyecto Final

Documentación académica del procesamiento, exploración y modelado del conflicto geopolítico.

Planteamiento del Problema Geopolítico

La estabilidad de Oriente Medio y el mercado energético mundial dependen críticamente de las dinámicas del conflicto entre Irán, Israel y los Estados Unidos. Este proyecto diseña un sistema de alerta analítico para clasificar diariamente el nivel de confrontación.

Pregunta de Machine Learning:
¿Es posible clasificar el nivel de escalada del conflicto Irán–Israel–EE.UU. en una ventana país-día usando señales mediáticas, lingüísticas, económicas y de eventos físicos de fuentes abiertas?

Fuentes OSINT Integradas

  • ACLED: 29,811 eventos físicos violentos georreferenciados.
  • GDELT: Tono diario de noticias y volumen de artículos globales.
  • Yahoo Finance: Precios del barril Brent como proxy económico.

Embeddings Lingüísticos

Procesamos 685 titulares de prensa con all-MiniLM-L6-v2, generando vectores de 384 dimensiones. La clasificación semántica usa similitud coseno contra plantillas de escalada pre-definidas, superando el sesgo de negación del conteo de palabras.

Análisis Exploratorio de Datos (EDA)

El dataset País-Día contiene 2,592 registros (864 días × 3 países) entre 2024-01-01 y 2026-05-13.

Desbalance del Target

52.5% Baja · 32.1% Media · 15.4% Alta. Requiere balanceo con SMOTE.

Volatilidad del Brent

Mayor dispersión y outliers en el precio del barril durante días de Alta Escalada (2).

Focalización Geográfica

ACLED concentra violencia física en Israel e Irán. EE.UU. se expresa principalmente en señal mediática.

Correlación de Variables: Los embeddings score_alta_prom y pct_alta_emb muestran correlación lineal positiva significativa con el target, mientras tono_gdelt presenta correlación negativa esperada.

Pipeline de Preprocesamiento y Validación

Pipeline secuencial reproducible para preparar el dataset para entrenamiento:

1
Codificación

One-Hot Encoding sobre la columna pais → 3 columnas binarias indicadoras.

2
Validación Temporal

Train (2024–2025: 2,193 registros) / Test (2026: 399 registros). Sin lookahead bias.

3
Estandarización

StandardScaler → media 0 y desviación 1 por variable. Clave para KNN y modelos lineales.

4
Balanceo SMOTE

Sobremuestreo sintético sobre el set de entrenamiento → 3,414 registros balanceados (1,138 por clase).

Mitigación de Target Leakage: Se excluyeron escalada_acled, tono_gdelt y sentimiento_medio del vector de descriptores (Escenario B), forzando al modelo a aprender patrones reales de señales brutas.
